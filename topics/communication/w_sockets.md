# WebSockets

What it is: A persistent, two-way connection between client and server.

How it works:
- One connection stays open
- Client and server can send messages anytime
- No request/response waiting

Pros
- True real-time
- Very efficient
- Bi-directional

Cons
- More complex
- Needs infrastructure support

Example use
- Chat apps
- Multiplayer games
- Live collaboration (Google Docs)
- Trading dashboards
